# Lark Agent 功能清单（基于高频场景）

**飞书排版说明**：全文为云文档友好结构（标题 + 列表，无 Markdown 表格）。复制到飞书云文档后可直接分段粘贴；若某段需要表格，可在飞书中选中文字使用「转为表格」。

---

**前置约束**

- 数据主要在**数仓**
- Agent 服务**业务群**
- 需要**对话问答** + **定时推送卡片**

---

## 一、场景 1：产品/策划问数（新增、留存等）

### 1.1 指标问答（受控）｜难度：中

- **具体能力**：支持自然语言问昨日/近 7 日新增、D1/D7 留存、活跃等，返回数字，可选环比/同比。
- **实现要点**：后端维护「指标 ID + 维度 + 时间范围」白名单；LLM 只做槽位填充，执行固定 SQL 或语义层。

### 1.2 维度下钻（有限）｜难度：中

- **具体能力**：按国家、渠道、平台、包体等已开放维度拆分。
- **实现要点**：维度枚举写死或来自配置表；禁止自由拼接未审批字段。

### 1.3 口径说明一键带出｜难度：低

- **具体能力**：回答末尾附指标定义摘要或「查看完整口径」链接。
- **实现要点**：绑定指标字典（文档/wiki 或结构化表）。

### 1.4 定时卡片对齐｜难度：中（工程纪律）

- **具体能力**：日报/周报卡片与对话用同一套结果口径。
- **实现要点**：与对话共用**同一套指标 API**，避免群里与报表不一致。

### 1.5 超出范围引导｜难度：低

- **具体能力**：识别要明细、用户级、未上线指标等，明确引导走工单或 BI。
- **实现要点**：规则 + 少量 prompt。

**MVP 建议**：先上 5～8 个最高频指标（新增、活跃、D1/D7 留存等）和 3～5 个维度；其余统一引导到工单。

## 二、场景 2：对报表里的数有疑问，需要解释

**说明**：本质是「解释 + 排错」，建议拆成文档型与数据型两条能力。

### 2.1 报表/指标词典问答（RAG）｜难度：中

- **具体能力**：例如报表里留存为何是 D1、为何与后台不一致，引用内部文档段落回答。
- **实现要点**：指标说明、报表注释、常见 FAQ 切块入库做向量检索；回答须带来源片段。

### 2.2 同环比/异常话术模板｜难度：低～中

- **具体能力**：根据查询结果生成可能原因清单（活动、版本、数据延迟、过滤条件等）。
- **实现要点**：规则引擎 + 模板，LLM 润色；不编造具体业务原因。

### 2.3 「对照查询」小工具｜难度：中

- **具体能力**：用户描述报表单元格或截图信息，按同名指标、同时间、同维度再算一遍。
- **实现要点**：用户需能描述筛选条件；或提供交互卡片选日期、维度后重跑。

### 2.4 数据新鲜度/分区说明｜难度：低

- **具体能力**：回答数更新到哪一天、延迟多久。
- **实现要点**：数仓元数据或固定配置。

### 2.5 升级人工｜难度：低

- **具体能力**：文档对不上或两次对照仍不一致时，生成结构化摘要并 @ 数据同学。
- **实现要点**：卡片中带指标、时间、维度、两次结果、链接。

**注意**：解释类回答中，**数值以数仓或 API 为准**；文档只说明怎么算、常见误解，避免 LLM 编造数字。

## 三、场景 3：具体拉数需求

### 3.1 需求结构化（工单助手）｜难度：低～中

- **具体能力**：把口语整理为指标列表、时间、粒度、维度、过滤、交付格式、用途。
- **实现要点**：LLM 填表，发到群卡片或多维表格/工单系统。

### 3.2 模板化拉数｜难度：中

- **具体能力**：常见需求对应模板 ID（如分渠道分日新增留存宽表）。
- **实现要点**：模板绑定受控 SQL 或 BI 导出任务。

### 3.3 权限内自助导出链接｜难度：中

- **具体能力**：生成 BI 已保存查询链接，或推送脱敏聚合 CSV 到私聊/文档。
- **实现要点**：按角色控制模板可见性。

### 3.4 复杂需求分流｜难度：低

- **具体能力**：识别多表 join、用户级明细、一次性定制等，不自动跑数，只生成工单。
- **实现要点**：关键词 + 分类模型或规则。

### 3.5 进度通知｜难度：视系统而定

- **具体能力**：工单状态变更推送到群或私聊（若接工单系统）。
- **实现要点**：Webhook 等对接。

**MVP 建议**：全自动拉数只覆盖少数已审批模板；其余一律结构化工单，避免数仓被 ad-hoc 拖垮和权限失控。

## 四、汇总：Agent 模块实现清单

1. **对话网关**：收 Lark 消息、鉴权、限流、会话上下文（可选）。
2. **意图路由**：区分查数、问口径、对报表有疑问、提拉数需求。
3. **指标查询服务**：语义层或参数化 SQL，与定时卡片共用。
4. **知识库（RAG）**：报表说明、指标字典、FAQ。
5. **工单/模板模块**：拉数需求结构化、模板执行、分流规则。
6. **定时任务**：日报卡片，可选数据就绪检测。
7. **观测与审计**：谁问了什么、跑了哪条查询、失败原因。

---

## 五、推荐落地顺序（对应三类场景）

1. **先做场景 1**：指标白名单问答 + 定时卡片（同一 API）。
2. **再做场景 2**：指标字典与报表说明 RAG + 数据新鲜度 + 对照查询（卡片选条件）。
3. **最后场景 3**：拉数需求结构化工单 + 2～3 个高频模板自动出结果。
